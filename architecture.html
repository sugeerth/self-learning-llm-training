Width > depth (at this scale)

Shallow-wide (4L × 512d) beat deep-narrow (10L × 320d) by ~74 PPL points despite similar parameter counts (11.3M vs 11.1M).

GQA ≈ MHA, faster

Full MHA gave only ~10 PPL improvement over GQA at this step count, but GQA's tok/s in baseline (9,346) is 22% faster than MHA (7,658).

Untied = wasted params

Untied embeddings 5× the param count (48M vs 9.4M) but did worse than tied baseline-class runs at 200 steps — undertrained.

Takeaways

What the sweep tells us

1 · Width > depth at small scale + short training

When you only get 200 steps, wider matrices push more useful signal per token than another residual stream. This flips at larger compute budgets — but holds here.

2 · GQA is a free lunch

GQA (n_kv=2) lost ~10 PPL vs full MHA but ran 22% faster on baseline. At the point KV cache memory dominates, GQA wins outright.

3 · Don't untie embeddings without compute

Untied embeddings 5× param count (48M) but val PPL got worse at fixed steps. The lm_head needs lots of gradient updates to be useful — tied head reuses tok_emb's training signal.

4 · Long context costs you at fixed step budget

Doubling context (128 → 256) at the same step count hurt loss — fewer effective gradient updates per token. Pay for context only if you can also pay for steps.
