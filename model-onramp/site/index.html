model-onramp

Models are plugins. Infrastructure is permanent.

model-onramp is the onboarding layer between LLM providers and everything you build on top of them. When a new model ships — a new Claude, an open-weights release, your own fine-tune — it enters through the on-ramp: registered, measured, routed, and finally trusted, all without touching the training loops, evals, judges, or dashboards that consume it.

The road in

  1. OnboardOne ~20-line adapter file — or none: onramp discover reads the live Anthropic Models API and registers new models by itself.
  2. MeasureBudget-capped probes test what config files only claim: JSON reliability, instruction following, tool calls, speed, usable context.
  3. RouteAgents ask for a role — “judge”, “trainer” — never a model name. Rankings double as failover chains, with retries and cost caps.
  4. LearnLive outcomes feed back in. Autopilot promotes models that earn trust, demotes ones that decay, and a breaker skips ones that are down.

Lifecycle

candidate→ stable→ retired

A freshly probed model is a candidate: routable, but ranked below every stable model, earning a small exploration share of live traffic. Once it clears the evidence bar — enough calls, high success rate, quality matching the stable cohort, confirmed pricing — autopilot promotes it to stable and it serves first. Models that decay are demoted; retired models never route.

What’s inside

Empirical manifests

Capabilities are measured, not declared — versioned, snapshotted, and diffed. Weekly CI re-probes catch silent model updates as drift.

Self-learning router

Circuit breaker, bandit exploration, retry-then-failover, per-session cost caps. Every decision lands in an event stream.

Judges as teachers

In the self-learning loop, the Judge’s verdict scores the Evaluator’s model and the MetaJudge scores the Judge’s — the hierarchy trains its own router.

Offline everything

Mock adapters pass or fail every probe, so the whole pipeline — probes, routing, autopilot — runs in CI with no API keys.

Quick start

# in self-learning-llm-training/model-onramp
pip install -e ".[dev]" && python -m pytest tests/ -q   # 42 passed, no keys needed

export ANTHROPIC_API_KEY=...
python -m onramp discover --probe    # auto-register + measure new models
python -m onramp resolve judge       # best model for a role + fallback chain
python -m onramp autopilot --apply   # promote/demote from live evidence
python -m onramp serve               # live dashboard on :8010

From application code, no model names anywhere:

from onramp import OnrampClient

client = OnrampClient(cost_cap_usd=5.00, explore_rate=0.1)
result = client.chat(messages, role="judge")   # routed + failover + tracked
client.feedback(result, score=0.9)             # teach the router
